# 初心者がGitHubのREADMEを作ってみた✏️

### 実際に私がつまづいた部分をまとめながらREADMEの作り方についてまとめていく📒
初心者だからこそわからない最初の疑問を一緒に解決していきましょう！

## まずREADMEとは何を書く場所なのか
 
  リポジトリを開いたときに一番最初に表示される場所。自分や他の人が見たときに「これは何のためのプロジェクトか」をひと目でわかるようにする場所。
  今回私は、学習のログを残し第三者から見て何を勉強したのかわかりやすいようにREADMEを書いていきます。
  
  **学習記録のリポジトリであれば、以下の3つに気を付けて書く**
  1. **「何」を記録しているか（概要）**  
      「自分が日々学んだことをアウトプットする場所です」といった簡単な説明。  
       例：# Today I Learned (TIL)  
       日々の学習内容（OS、ミドルウェア、クラウドなど）を記録しています。  

  2. **「どうやって」記録しているか（ルール）**  
      自分で決めたフォルダ構造のルールなどを書いておくと、後で迷いません。  
      例：## 構造    
      YYYY-MM/YYYY-MM-DD.md の形式で保存します。  
  3. **「どこに」何があるか（インデックス・目次）**  
     「階層構造のリンク」を載せる場所！  
      READMEに最新の月へのリンクを貼っておけば、いちいちフォルダを辿らなくても、クリック一発で今日の学習記録に飛べます。

 **この３つを意識しながら書くといいです！**
